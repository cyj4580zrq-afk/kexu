# 关于学生课表辅助应用“课序”试用及教务系统访问方式的说明

尊敬的武汉纺织大学外经贸学院信息中心老师：

您好！

我是本校学生________（学号：________，学院/班级：________）。为了方便同学查看个人课程安排，我利用课余时间开发了一款名为“课序”的移动端课程表工具。现就软件用途、功能、教务系统访问方式及个人信息保护措施向贵中心作出说明，并申请对软件的合规使用方式给予指导。

## 一、开发目的与使用范围

“课序”定位为面向本校学生的非官方、非商业课程表辅助工具，不收取费用，不含广告，不进行商业推广，也不代表学校或学校任何部门。软件只帮助学生整理本人有权查看的课程信息，不提供成绩修改、选课操作、他人信息查询等功能。

在得到学校明确意见前，我将控制使用范围，不进行大规模推广。如学校认为某项功能或访问方式不合适，我会立即停止相关功能并配合整改。

## 二、主要功能

1. 学生使用本人教务系统账号，在本人设备上同步个人课表；
2. 按周次显示课程，并支持周课表网格和课程列表两种视图；
3. 展示课程名称、教师、地点、节次、具体时间、上课周次及结课状态；
4. 在教务系统夜间维护或暂时无法访问时，继续查看上一次成功同步的本地课表和同步快照；
5. 支持手动添加、删除课程，设置当前教学周、课程时长、主题和显示方式；
6. 提供 Android 版本，并保留 iOS 工程，二者采用相同的数据处理原则。

## 三、教务系统访问方式

课表同步只在学生主动点击“登录并同步课表”后发生。请求由学生手机直接通过 HTTPS 连接学校教务系统，不经过开发者服务器，不设中转代理，也不建立教务账号数据库。

软件只使用当前学生本人提供的账号完成正常登录，并请求本人课表页面所需的数据。软件不会批量抓取，不会遍历账号，不会绕过验证码、访问控制或其他安全机制，也不会查询与课程表无关的数据。若教务系统要求验证码、二次认证或拒绝访问，软件将停止同步并提示用户，不尝试规避限制。

为避免增加学校系统负担，软件不进行后台轮询或自动高频同步；每次同步均由用户手动发起。希望贵中心能够告知是否有学校认可的接口、登录方式、请求频率或维护时段要求，我会按要求调整实现。

## 四、个人信息与隐私保护

软件遵循数据最小化原则：

1. 教务系统密码仅在当次登录过程中使用，并在传输前按教务系统现有方式加密；密码不会写入手机持久化存储，也不会上传给开发者；
2. 学号仅保存在学生当前设备，用于下次填写时提供便利，不上传至开发者服务器；
3. 课表、课程编辑结果及同步快照只保存在学生当前设备，不进行云端备份或集中收集；
4. 软件不集成广告 SDK、用户行为统计 SDK或第三方追踪服务；
5. 软件只申请联网和必要的触感反馈权限，不申请通讯录、短信、相册、定位等与功能无关的权限；
6. 用户可在设置中清空课程，卸载软件后可删除全部应用本地数据；
7. 软件界面明确标注其为学生个人开发的非官方工具，并在首次同步前展示隐私说明、取得用户确认。

## 五、源代码与安全配合

项目源代码目前公开托管于 GitHub，便于检查功能实现和数据流向。如贵中心需要，我愿意提交源代码、安装包、网络请求说明及测试账号环境下的操作演示，配合开展安全审查。若发现安全风险，我会第一时间下架相关版本、修复问题并通知使用者更新。

## 六、希望贵中心指导的事项

1. 该软件在上述范围内供本校学生自愿使用，是否符合学校信息系统管理要求；
2. 学校是否提供或计划提供经授权的个人课表查询接口；
3. 对账号登录、请求频率、数据字段、软件名称和非官方标识是否有具体要求；
4. 在正式提供给同学使用前，是否需要办理备案、授权、安全测试或签署相关承诺；
5. 贵中心认为需要补充或调整的其他安全及隐私措施。

感谢老师审阅。开发这款工具的出发点是减少同学整理和查看课表的不便，同时尊重学校系统安全、学生个人信息和学校管理要求。我愿意按照贵中心意见完善软件，并在未获认可或存在风险时停止相关教务同步功能。
