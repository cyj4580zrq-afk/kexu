# 更新日志

本项目的每个本地版本都会在这里记录主要调整。自 0.5.4 起，安装包仅在本机生成，不自动上传 GitHub。

## [0.9.2stable] - 2026-07-21

- 成绩按学期缓存：已保存的学期直接显示本地数据，不再重复请求教务系统。
- 成绩页支持删除当前选中学期的全部本地成绩；删除后可重新查询该学期。设置页新增 QQ 反馈群 `1075730072`。
- 修正缓存匹配：改按教务系统学年和学期编码识别已保存成绩，避免显示文本差异导致同学期被重复查询。
- 成绩查询支持分页拉取，按教务系统总条数保存完整学期成绩；课程详情可解析并展示平时、期末等分项成绩与占比。
- 分页请求使用教务页面的 15 条尺寸并识别重复页，避免翻页将同一批成绩重复追加；分项成绩详情增加 POST 回退请求与重试。
- 以表单编码请求成绩详情；移除不兼容的分页字段，恢复成绩查询。
- 成绩分页改为教务系统实际使用的 `queryModel.showCount` 与 `queryModel.currentPage` 参数，支持读取完整学期成绩。
- 成绩分项详情改为教务系统实际的 POST 请求，带 `time`、`gnmkdm` 和表单参数，读取平时、期末等成绩及占比。
- 详情请求对齐教务系统的 HTTP 地址与 Referer；读取失败时显示实际错误原因以便定位。
- Android 仅为 `jw.whcibe.com` 放行教务系统所需的 HTTP 详情接口，修复分项成绩请求被系统网络策略拦截的问题。

## [0.8.1] - 2026-07-21

- 优化预测返回掉帧：手势期间暂停页面内部的玻璃模糊，取消逐帧圆角和阴影变化，让返回过程以合成层位移为主；结束后自动恢复全局液态玻璃。
- 一级页面切换新增短促的上浮淡入动画；底部抽屉和详情弹窗继续保留各自的进入动效，避免界面硬切。

## [0.8.0] - 2026-07-21

- 液态玻璃升级为全局主题：课程、成绩、记录、课表、设置、输入控件、弹窗、底部导航和操作按钮统一使用半透明模糊材质。
- 设置中的“全局液态玻璃”开关可一键启用或关闭全部玻璃效果；默认关闭以兼顾低性能设备与阅读清晰度。
- 保留课程状态、成绩分数和危险操作的颜色层级，避免全局玻璃化降低信息辨识度。

## [0.7.4] - 2026-07-21

- 优化预测返回性能：手势进度不再触发 Vue 全页面响应式渲染，改为直接更新 CSS 变量；课程和设置内容较多时，连续返回仍保持跟手。
- “全透明底部玻璃”改为真正透明背景，保留背景模糊、边缘高光和阴影以维持玻璃层次；开启时自动启用底部导航玻璃。

## [0.7.3] - 2026-07-21

- 移除页面由隐藏状态恢复时的通用淡入动画。预测返回过程只保留当前页的连续手势位移，避免首页首帧透明而露出底色，消除返回完成后的白闪。

## [0.7.2] - 2026-07-21

- 重构预测返回页面层级：不再创建或销毁临时首页预览，手势期间直接露出已常驻的真实首页；当前二级页面单独滑出，消除返回完成时的页面替换闪屏。

## [0.7.1] - 2026-07-21

- 修正预测返回完成后的预览交接：真实首页先完成绘制，再移除临时预览，避免返回后首页被重新显现而闪屏。
- 设置新增“全透明底部玻璃”选项；开启时自动启用底部导航玻璃，以更低不透明度、背景模糊和高光边缘呈现悬浮导航。

## [0.7.0] - 2026-07-21

- 底部五栏重绘为悬浮胶囊导航：圆角实体底座、清晰阴影和带圆形承托的当前入口，提升单手浏览时的识别度。
- 设置新增“底部导航玻璃”独立开关；开启后仅为底部导航应用透明、背景模糊和高光边缘，不再与全局液态玻璃强绑定。
- 调整页面底部留白及添加课程按钮高度，避免悬浮导航遮挡列表内容或操作入口。

## [0.6.5] - 2026-07-21

- 一级页面改为常驻显示切换，不再在返回首页时销毁并重新创建首页内容，修正返回后页面刷新、闪屏与状态丢失。
- 首页、全部课表、同步、成绩、记录和设置均会保留当前实例；从二级页面返回时直接显示原有页面。

## [0.6.4] - 2026-07-21

- 修正预测返回提交后首页抖动显现：返回完成时强制页面容器立即回到正常尺寸与位置，避免移除手势样式后再次触发默认过渡。

## [0.6.3] - 2026-07-21

- 原生层合并预测返回拖动进度：每个屏幕帧最多向 WebView 发送一次最新进度，减少快速拖动时的回调堆积与卡顿。
- 返回取消或完成时立即作废排队中的旧进度，防止页面已回到首页后又被过期手势事件拉动，消除二次回弹。

## [0.6.2] - 2026-07-20

- 修正 Android 预测返回与 Capacitor 普通返回回调同时处理同一手势的问题，避免当前页尚未滑离时首页被提前切换。
- 原生预测返回回调改为优先接管，启用手势返回动效时不再由 Capacitor 重复提交页面返回。

## [0.6.1] - 2026-07-20

- 修正手势返回不跟手：手势进行阶段不再使用页面默认缓动，当前页会严格按系统手势进度移动。
- 修正返回完成后首页二次回弹：首页在手势提交后的首次渲染取消进入动画，避免已经返回的页面再次上浮。

## [0.6.0] - 2026-07-20

- 重做 Android 预测返回提交过程：当前页会先连续滑离，再显示上一页，避免松手时页面被突然替换或覆盖；取消手势只回弹当前页，不会卸载页面。
- 设置新增“手势返回动效”开关，默认关闭；关闭时保留普通返回逻辑。
- 设置新增“液态玻璃”开关，默认关闭；开启后为底部导航、抽屉、确认弹窗和分段控件应用半透明材质、背景模糊与高光边缘。
- 下拉回弹默认关闭，可由用户按偏好启用。

## [0.5.9] - 2026-07-20

- 导航栏、底部抽屉、退出确认和分段操作控件加入克制的液态玻璃材质，使用半透明、背景模糊和细高光边框强化层级。
- 课程卡片、成绩列表和首页主课程卡仍保留高对比实体表面，确保文字、状态颜色和长列表在浅色与深色模式下清晰易读。
- 调整玻璃控件在深色模式下的透明度与阴影，避免发灰、泛白或影响底部导航的识别。

## [0.5.8] - 2026-07-20

- 修正预测返回动画变量：不再依赖 Android WebView 可能忽略的 CSS 乘法计算，缩放、位移、圆角和遮罩透明度均由前端预计算。
- 修正 Vue 挂载容器不处理动态样式的问题，预测返回进度直接写入页面根节点，确保动画变量可传递到全部页面与浮层。
- 原生预测返回回调调整为 Activity 恢复后注册，降低与 Capacitor 返回处理器的回调优先级冲突风险。

## [0.5.7] - 2026-07-20

- 接入 Android 预测返回手势：从边缘滑动时，当前二级页面会跟随手势缩放、圆角化并预览首页；松手取消时平滑回弹。
- 底部抽屉、课程与成绩详情、学期选择、退出确认等浮层支持跟手缩小并在返回完成后关闭。
- Android 13 使用标准返回提交，Android 14 及以上读取预测返回进度；仍复用原有“关闭浮层、返回首页、确认退出”规则，避免重复触发。

## [0.5.6] - 2026-07-19

- 直接读取 Android 状态栏、挖孔和底部导航栏的实际安全区，并传入页面 CSS；顶部内容、提示消息和五栏底部导航均依据真实设备尺寸避让。
- 解决部分 Android 15/16 系统未向 WebView 提供安全区变量时，页面仍压住状态栏或底部手势区的问题。

## [0.5.5] - 2026-07-19

- Android 原生层为系统状态栏预留内容安全区，避免首页、同步页和设置页的顶部标题与状态栏重叠。
- 重绘顶部提示消息：在窄屏自动收窄、换行并避开状态栏，不再被遮挡或横向溢出。
- 针对矮屏设备压缩首页与二级页的顶部留白和卡片高度，保留底部导航的安全区域。

## [0.5.4] - 2026-07-19

- 课表同步与成绩查询拆分为独立入口：同步课表不会查询或改动成绩，查询成绩不会改动课表。
- 成绩按学期合并保存到本机；“全部”旁可切换已查询学期，重新查询只更新该学期。
- 成绩卡片与详情新增任课教师；成绩详情请求补齐教务系统参数，并保留服务端返回的平时、期中、期末、实验等明细。
- 优化小屏布局、横向学期栏与弹层层级，避免窄屏出现遮挡或无法完整显示。

## [0.5.3] - 2026-07-19

- 新增“成绩”页：按学期查看课程总评、学分、绩点与成绩性质。
- 同一次教务登录会话会同步课表与学生个人成绩；成绩数据及同步快照只保存在当前设备。
- 单科成绩详情按需请求，支持展示教务系统提供的平时、期中、实验、期末、作业和占比等成绩组成。
- 成绩接口暂时不可用时，已同步的课表、成绩和历史快照都会保留。

## [0.5.2] - 2026-07-19

- 新增原生 iOS XCUITest 冒烟测试，自动检查首页、完整课表、课程添加、同步隐私说明和设置页面。
- iOS GitHub Actions 会自动选择并启动可用的 iPhone Simulator，在干净环境中执行界面交互测试。
- 每次测试都会上传页面截图、Xcode 测试结果包、构建输出和应用日志，失败时也能保留诊断材料。
- 版本号更新至 `0.5.2`，Android Release 继续由版本标签自动生成正式签名 APK。

## [0.5.1] - 2026-07-19

- Android 安装包改为正式 Release 构建，使用长期发布密钥签名，不再发布 Debug APK。
- GitHub Releases 自动构建流程接入加密签名密钥，版本标签会生成可直接安装的正式 APK。
- 教务同步增加首次隐私确认，明确应用为非官方学生工具，并说明密码、学号、课表和同步记录的数据处理方式。
- 设置页新增“隐私与非官方说明”入口，版本号同步更新至 `0.5.1`。

## [0.5.0] - 2026-07-19

- 新增完整的 Capacitor iOS 原生工程，与 Android 共用课程表、教务同步、本地存储、深色模式和课程时间设置。
- 配置 iOS 应用名称“课序”、应用标识 `com.campusflow.schedule`、统一图标与 `0.5.0` 版本信息。
- 增加 iOS 同步和 Xcode 构建命令，并完善 Pods、签名文件与构建产物的忽略规则。
- 将移动端工程统一命名为 `kexu-mobile`，版本页不再只显示 Android 平台。

## [0.4.1] - 2026-07-19

- 新增“每小节时长”设置，默认 45 分钟，可按 5 分钟在 30–60 分钟之间调整。
- 每两节大课内部保留 5 分钟小课间，上午第 2 节和下午第 6 节结束后保留 20 分钟长课间。
- 调整默认节次时间：1–2 节 08:15 开始，5–6 节 13:30 开始，9–10 节为 17:45–19:20，11–12 节为 19:30–21:05。
- 课程详情新增实际起止时间，首页上课状态判断与周课表时间轴会随课时长度同步更新。

## [0.4.0] - 2026-07-19

- 新增七天十二节的周课表网格，可直观看到课程时间与位置。
- 保留按星期分组的课程列表，支持通过右上角按钮切换两种课表布局。
- 自动记住用户选择的课表布局。
- 重绘教务同步页面的学期选择界面，改为适合手机操作的底部面板。
- 优化 360px 与 390px 手机屏幕适配，并继续突出显示已结课课程。
